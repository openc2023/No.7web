# 修复画布布局问题，实现完全填满模式

## 1. 分析当前问题
- `.canvas-scroller` 有 40px 的内边距，导致画布周围有灰色边距
- `.canvas-frame` 设置了 `max-width: 1200px`，导致在高分辨率显示器上右侧有黑色空隙
- GrapesJS 编辑器的宽度和高度可能没有设置为 100%

## 2. 实施方案 A：完全填满模式
### 2.1 修改 CSS 样式
- 修改 `.canvas-scroller`：去掉内边距和居中对齐，设置黑色背景
- 修改 `.canvas-frame`：去掉最大宽度限制，去掉阴影，设置高度至少填满
- 添加 `.gjs-cv-canvas` 样式：确保 GrapesJS 容器也填满

### 2.2 检查 JavaScript 配置
- 确保 `initGrapesJS()` 函数中 `height` 和 `width` 设置为 `'100%'`

## 3. 实施步骤
1. 找到并替换 `.canvas-scroller` 的 CSS 样式
2. 找到并替换 `.canvas-frame` 的 CSS 样式
3. 添加 `.gjs-cv-canvas` 的 CSS 样式
4. 检查并修改 `initGrapesJS()` 函数中的配置
5. 测试效果，确保画布完全填满黑色区域

## 4. 预期效果
- 白色画布会占满整个黑色区域，没有灰色边距
- 没有右侧的黑色空隙
- 界面变得饱满，符合用户的"填满"要求
